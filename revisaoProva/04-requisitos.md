# 🔹 Requisitos de Software

No contexto da engenharia de software, os requisitos representam a base de todo o desenvolvimento. São eles que descrevem **o que o sistema deve fazer** e **como ele deve se comportar**, servindo como referência para todas as demais etapas do projeto.

Um erro na definição de requisitos pode comprometer todo o sistema, gerando retrabalho, aumento de custos e insatisfação do usuário. Por isso, compreender corretamente esse conceito é essencial.

---

## 📌 O que são Requisitos?

Requisitos são descrições das necessidades, expectativas e restrições relacionadas a um sistema de software.

Eles podem ser vistos como um **acordo entre stakeholders (clientes, usuários, equipe técnica)** sobre o que será desenvolvido.

De forma geral, os requisitos respondem a duas perguntas principais:

- **O que o sistema deve fazer?**
- **Como o sistema deve se comportar?**

---

## ⚙️ Requisitos Funcionais

Os requisitos funcionais descrevem as **funcionalidades do sistema**, ou seja, as ações que ele deve ser capaz de executar.

Eles estão diretamente relacionados ao comportamento do sistema e às interações com o usuário.

### 📌 Características

- Representam serviços ou funções do sistema  
- São diretamente perceptíveis pelo usuário  
- Estão ligados às regras de negócio  

### 🧾 Exemplos

- O sistema deve permitir que o usuário realize login  
- O sistema deve calcular o total de uma compra  
- O sistema deve permitir cadastrar, editar e excluir produtos  
- O sistema deve gerar um relatório de vendas  

Esses requisitos são essenciais para garantir que o sistema cumpra seu propósito principal.

---

## 🛠️ Requisitos Não Funcionais

Os requisitos não funcionais descrevem **características de qualidade do sistema**, ou seja, como o sistema deve operar.

Eles não definem o que o sistema faz, mas sim **como ele deve se comportar em termos de desempenho, segurança, usabilidade, entre outros aspectos**.

### 📌 Características

- Relacionados à qualidade do sistema  
- Nem sempre são visíveis diretamente pelo usuário  
- Influenciam a experiência e a eficiência  

### 🧾 Exemplos

- O sistema deve responder a requisições em até 2 segundos  
- O sistema deve garantir autenticação segura dos usuários  
- O sistema deve estar disponível 99% do tempo  
- O sistema deve ser compatível com dispositivos móveis  

---

## ⚖️ Diferença entre Funcionais e Não Funcionais

Uma forma simples de diferenciar:

- **Requisitos funcionais** → descrevem *o que o sistema faz*  
- **Requisitos não funcionais** → descrevem *como o sistema faz*  

Ambos são igualmente importantes. Um sistema pode ter todas as funcionalidades corretas, mas falhar por ser lento, inseguro ou difícil de usar.

---

## 🔗 Relação com o Processo de Desenvolvimento

Os requisitos são definidos principalmente na fase de **análise**, mas influenciam todas as etapas do ciclo de software:

- No **projeto**, ajudam a definir a arquitetura  
- No **desenvolvimento**, orientam a implementação  
- Nos **testes**, servem como base para validação  
- Na **manutenção**, ajudam a identificar o que precisa ser ajustado  

Em metodologias ágeis, como o Scrum, os requisitos são organizados em um **Product Backlog**, geralmente representados por histórias de usuário.

---

## ⚠️ Problemas Comuns

Alguns problemas frequentes relacionados a requisitos incluem:

- Requisitos ambíguos ou mal definidos  
- Falta de comunicação com o cliente  
- Mudanças não controladas  
- Falta de priorização  

Esses problemas podem gerar atrasos e retrabalho, reforçando a importância de uma boa definição e gestão dos requisitos.

---

## 🧠 Conclusão

Os requisitos de software são o ponto de partida para qualquer sistema. Eles definem o que será construído e estabelecem as expectativas sobre o funcionamento e a qualidade do software.

Compreender a diferença entre requisitos funcionais e não funcionais é essencial, assim como entender seu papel dentro do processo de desenvolvimento.

Mais do que documentos formais, os requisitos representam a **ponte entre o problema do mundo real e a solução tecnológica**.
